# DateMate AI Lite - PRD (Product Requirements Document)

> 버전: 1.0 Lite
> 작성일: 2026-03-10
> 작성자: 김지욱

---

## 1. 제품 개요

### 1-1. 한 줄 요약
**"오늘 뭐하지?"를 3분 안에 해결해주는 랜덤 데이트 코스 생성기**

### 1-2. 제품 비전
데이트 계획을 세우는 데 드는 시간과 스트레스를 제거한다. 사용자가 직접 고르는 대신 랜덤 뽑기로 조건을 정하면, AI가 실제 존재하는 장소 기반으로 완성된 코스를 즉시 만들어준다.

### 1-3. 왜 Lite인가
- 핵심 가설 검증에 집중: "랜덤 뽑기 → AI 코스 생성" 플로우에 사용자가 반응하는가?
- 불필요한 복잡도 제거 (모드 분기, 블라인드 모드, 퀘스트, 친구 시스템 등)
- 빠른 출시 → 피드백 수집 → 풀 버전 방향 결정

---

## 2. 타겟 사용자

### 2-1. 핵심 타겟
**20-30대 커플 또는 친구 그룹**
- 매주 "이번 주말 뭐하지?" 고민을 반복하는 사람
- 네이버/인스타에서 맛집 검색하다가 30분 이상 소비하는 사람
- 새로운 동네, 새로운 경험을 원하지만 직접 찾기는 귀찮은 사람

### 2-2. 핵심 페인포인트
| 페인포인트 | 현재 해결 방식 | 문제점 |
|-----------|-------------|--------|
| 어디 갈지 모르겠다 | 네이버/인스타 검색 | 시간 소모, 선택 피로 |
| 맨날 같은 곳만 간다 | 지인 추천 | 범위 한정적 |
| 코스를 짜는 게 귀찮다 | 블로그 코스 따라가기 | 내 취향과 안 맞음 |
| 새로운 걸 하고 싶다 | 랜덤 도전 (유튜브 등) | 실행 방법이 없음 |

### 2-3. 사용 시나리오
> 토요일 오후, 커플 A와 B는 "오늘 뭐하지?"로 30분째 카톡 중이다. A가 DateMate를 열고 랜덤 뽑기를 돌린다. 2호선 → 성수 → 일식 → 내가 쏜다 → 바삭한 디저트 → 전시회. 30초 만에 결정되고, "코스 만들기"를 누르면 AI가 성수 주변 실제 맛집/카페/전시 정보로 코스를 짜준다. 지도에서 동선을 확인하고 바로 출발한다.

---

## 3. 핵심 플로우

```
앱 진입 → 랜덤 뽑기 (6단계) → 결과 확인 → AI 코스 생성 → 코스 상세 → 지도 보기
```

총 소요 시간 목표: **진입부터 코스 확인까지 3분 이내**

## 4. 기능 명세

### 4-1. 온보딩 (첫 진입 시 1회)

**의도:** 최소한의 정보로 AI 코스 품질을 높이기 위함. 가입 허들을 최대한 낮춘다.

| 항목 | 필수 여부 | 설명 |
|------|---------|------|
| 닉네임 | 필수 | 앱 내 표시용 (2-8자) |
| 위치 (시/구) | 필수 | 코스 추천 기준 지역 |
| 취향 태그 | 선택 | 좋아하는 키워드 3-5개 선택 (맛집, 카페, 자연, 액티브 등) |

- 가입 방식: 카카오 소셜 로그인 (1차), 이메일 (2차)
- 온보딩 화면 수: 최대 3장
- 스킵 가능 여부: 취향 태그만 스킵 가능

---

### 4-2. 홈 화면

**의도:** 앱을 열면 바로 핵심 행동(랜덤 뽑기)으로 유도한다. 부가 정보는 최소화.

**구성 요소:**

```
┌─────────────────────────────┐
│  안녕, {닉네임}님              │
│  오늘 어디로 떠나볼까?          │
│                              │
│  ┌─────────────────────────┐ │
│  │    🎲 랜덤 뽑기 시작      │ │
│  │    탭해서 오늘의 코스 뽑기   │ │
│  └─────────────────────────┘ │
│                              │
│  최근 코스 ──────── 전체보기 > │
│  ┌───────┐ ┌───────┐        │
│  │ 코스1  │ │ 코스2  │        │
│  └───────┘ └───────┘        │
└─────────────────────────────┘
```

- **메인 CTA**: "랜덤 뽑기 시작" 버튼 (화면의 50% 이상 차지)
- **최근 코스**: 이전에 생성한 코스 최대 4개 (가로 스크롤)
- 빈 상태: 코스가 없으면 "첫 번째 랜덤 코스를 뽑아보세요" 안내

---

### 4-3. 랜덤 뽑기 (가챠)

**의도:** 데이트 계획의 "결정"을 게임화한다. 직접 고르는 부담 없이, 랜덤의 재미로 결정을 대신해준다.

**6단계 뽑기 플로우:**

| 순서 | 항목 | 선택지 예시 | 의도 |
|------|------|-----------|------|
| 1 | 지하철 호선 | 1호선 ~ 9호선, 경의중앙선 등 | 대략적 위치 범위 결정 |
| 2 | 역 | 해당 호선의 역 목록 | 정확한 출발/활동 거점 결정 |
| 3 | 식사 종류 | 한식, 중식, 일식, 양식, 분식 등 | 메인 식사 카테고리 |
| 4 | 결제자 | 내가 쏜다, 네가 쏴, 더치페이, 랜덤 | 재미 요소 + 결제 결정 |
| 5 | 카페 키워드 | 바삭, 달콤, 씁쓸, 시원, 고소 등 | 카페/디저트 분위기 결정 |
| 6 | 놀거리 | 산책, 전시회, 영화, 술집, 공방 등 | 메인 액티비티 결정 |

**UX 상세:**
- 각 단계: 슬롯머신 애니메이션 (세로 스크롤 회전)
- "뽑기!" 버튼 탭 → 1-2초 회전 → 감속 → 결과 표시
- 결과가 미리 결정되고, 애니메이션이 그 값에 자연스럽게 멈춤
- 각 단계 완료 시 하단에 태그로 누적 표시
- 전 단계 완료 후 → 결과 요약 카드 + "이 조합으로 코스 만들기" 버튼
- "다시 뽑기" 버튼으로 전체 리셋 가능

**기술 의존성:**
- 순수 프론트엔드 (API 불필요)
- 지하철 노선/역 데이터: 정적 JSON

### 4-4. AI 코스 생성

**의도:** 랜덤 뽑기 결과를 기반으로, 실제 존재하는 장소들로 구성된 실행 가능한 코스를 만든다.

**입력 (뽑기 결과):**
```json
{
  "station": "성수",
  "meal": "일식",
  "cafeKeyword": "바삭",
  "activity": "전시회",
  "payer": "내가 쏜다"
}
```

**출력 (코스):**
```json
{
  "title": "성수동 감성 투어",
  "description": "성수역 주변 일식 맛집부터 전시회까지",
  "totalDuration": 240,
  "totalEstimatedCost": 65000,
  "stops": [
    {
      "order": 1,
      "name": "스시소라 성수점",
      "category": "일식",
      "address": "서울 성동구 성수이로 ...",
      "latitude": 37.xxx,
      "longitude": 127.xxx,
      "description": "오마카세 런치가 가성비 좋은 스시집",
      "recommendedMenus": ["런치 오마카세", "사케"],
      "estimatedTime": 70,
      "estimatedCost": 25000,
      "imageUrls": ["..."]
    },
    { "order": 2, "...": "카페 (바삭한 디저트)" },
    { "order": 3, "...": "전시회/갤러리" },
    { "order": 4, "...": "산책 또는 부가 스팟" }
  ]
}
```

**생성 로직:**
1. 뽑기 결과의 역 이름 → 해당 지역구 매핑 (예: 성수 → 성동구)
2. 네이버 검색 API로 "{역 이름} {식사종류} 맛집", "{역 이름} 카페", "{역 이름} {놀거리}" 검색
3. 한국관광공사 TourAPI로 해당 지역 관광지/맛집 조회
4. 검색 결과 + 사용자 취향 태그 → Gemini AI에 코스 생성 요청
5. AI 응답 파싱 → 각 장소별 네이버 이미지 검색으로 사진 보강
6. 완성된 코스 데이터 반환

**코스 구성 규칙:**
- 3-5개 장소로 구성
- 식사 → 카페 → 놀거리 순서 기본 (AI가 자연스럽게 조정)
- 도보 이동 가능한 동선으로 배치
- 각 장소에 예상 소요시간, 1인 예상 비용 포함
- 실제로 존재하는 장소만 사용 (AI 생성 가게 금지)

**에러 처리:**
- 네이버 API 실패 → TourAPI 결과만으로 생성
- 둘 다 실패 → "해당 지역 정보를 찾을 수 없어요" + 다시 뽑기 유도
- AI 응답 파싱 실패 → 재시도 1회 → 실패 시 에러 메시지

**로딩 UX:**
- 생성 중 (10-20초 예상): 로딩 애니메이션 + "코스 만드는 중..." 텍스트
- 단계별 진행 표시: "장소 검색 중..." → "코스 구성 중..." → "거의 다 됐어요!"

### 4-5. 코스 상세 화면

**의도:** 생성된 코스의 전체 정보를 한눈에 보여준다. 각 장소의 상세 정보를 확인하고 바로 실행에 옮길 수 있게 한다.

**구성 요소:**

```
┌─────────────────────────────┐
│  ← 뒤로                     │
│                              │
│  [히어로 이미지]              │
│                              │
│  성수동 감성 투어              │
│  약 4시간 · 4곳 · 약 6.5만원   │
│  #일식 #바삭디저트 #전시회      │
│                              │
│  ── 코스 타임라인 ──          │
│                              │
│  ① 스시소라 성수점     70분    │
│     일식 · 약 2.5만원          │
│     [사진] [사진]             │
│     추천: 런치 오마카세         │
│     ↓ 도보 5분                │
│                              │
│  ② 어니언 성수        50분    │
│     카페 · 약 8천원            │
│     [사진] [사진]             │
│     추천: 크루아상, 라떼        │
│     ↓ 도보 10분               │
│                              │
│  ③ 대림창고 갤러리     60분    │
│     전시 · 약 1.5만원          │
│     ↓ 도보 3분                │
│                              │
│  ④ 서울숲             60분    │
│     산책 · 무료               │
│                              │
│  ┌─────────────────────────┐ │
│  │   🗺️ 지도에서 코스 보기   │ │
│  └─────────────────────────┘ │
└─────────────────────────────┘
```

**각 장소 카드 정보:**
- 장소명, 카테고리, 주소
- 사진 (최대 3장, 가로 스크롤)
- 추천 메뉴 (식당/카페인 경우)
- 예상 소요시간, 1인 예상 비용
- 다음 장소까지 도보 이동 시간

**인터랙션:**
- 장소명 탭 → 네이버 지도 앱/웹으로 이동 (길찾기)
- "지도에서 코스 보기" → 지도 화면으로 전환
- 저장 버튼 → 로컬 저장 (나중에 다시 보기)

### 4-6. 지도 화면

**의도:** 코스의 전체 동선을 시각적으로 확인한다. 각 장소의 위치 관계와 이동 경로를 직관적으로 파악할 수 있게 한다.

**구성 요소:**
- 네이버 지도 SDK 기반 전체 화면 지도
- 각 장소를 번호 마커로 표시 (①②③④)
- 마커 간 점선으로 경로 연결
- 하단 시트: 현재 선택된 장소 요약 카드
- 마커 탭 → 해당 장소 정보 하단 시트에 표시

**지도 초기 뷰:**
- 모든 마커가 보이도록 자동 줌/바운드 조정
- 첫 번째 장소에 포커스

**하단 시트 구성:**
- 장소명, 카테고리, 예상 시간
- "네이버 지도에서 열기" 버튼 (딥링크)
- 좌우 스와이프로 이전/다음 장소 전환

---

### 4-7. 코스 목록 (히스토리)

**의도:** 이전에 생성한 코스를 다시 볼 수 있게 한다. 저장 기능으로 마음에 드는 코스를 관리한다.

**구성:**
- 생성 순서대로 정렬 (최신순)
- 각 코스: 제목, 장소 수, 소요시간, 예상비용, 태그
- 저장된 코스 필터링 가능
- 코스 탭 → 코스 상세 화면으로 이동

## 5. 기술 스택

| 영역 | 기술 | 선택 이유 |
|------|------|---------|
| 프레임워크 | Next.js 14 (App Router) | SSR + 정적 생성 혼합, Vercel 배포 최적화 |
| 스타일링 | Tailwind CSS | 빠른 UI 개발, 일관된 디자인 시스템 |

| DB | Supabase PostgreSQL | 코스 저장, 유저 데이터 |
| AI | Google Gemini API | 코스 생성 (구조화된 JSON 출력) |
| 장소 검색 | 네이버 검색 API | 한국 장소 데이터 최적 |
| 관광 데이터 | 한국관광공사 TourAPI | 무료, 공공 관광 데이터 |
| 지도 | 네이버 지도 SDK | 한국 지도 최적, 마커/경로 지원 |
| 배포 | Vercel | Next.js 네이티브 지원, 자동 배포 |

## 7. Lite에서 제외하는 기능

| 기능 | 제외 이유 |
|------|---------|
| 모드 분기 (혼놀/커플/친구) | 복잡도 증가, 핵심 가설과 무관 |
| 블라인드 모드 | 부가 기능, 코어 플로우 검증 후 추가 |
| AI 채팅 (코스 상담) | 별도 기능, Lite 범위 초과 |
| 친구 시스템 (초대/연인 설정) | 소셜 기능은 검증 후 추가 |
| 퀘스트/미션 | 게이미피케이션은 리텐션 확보 후 |

## 10. 리스크 및 대응

| 리스크 | 영향 | 대응 |
|--------|------|------|
| AI 코스에 폐업된 가게 포함 | 신뢰도 하락 | 네이버 API 실시간 검색으로 최소화 + "정보가 다를 수 있어요" 고지 |
| 코스 생성 시간이 너무 김 (20초+) | 이탈 | 단계별 로딩 UX + 백그라운드 캐싱 |
| 특정 역 주변에 장소가 부족 | 빈약한 코스 | 인접 역까지 검색 범위 확장 |
| API 비용 초과 | 서비스 중단 | 일일 생성 횟수 제한 (비로그인 3회, 로그인 10회) |
| 저작권 (네이버 이미지) | 법적 이슈 | 이미지 직접 저장 안 함, URL 참조만 사용 + 출처 표기 |
